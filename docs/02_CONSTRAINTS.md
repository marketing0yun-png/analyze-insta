# 02. 실현가능성 / 제약 매트릭스 ⭐ 가장 중요

> 이 문서가 프로젝트의 **현실 경계**다. UI·기능 설계 전에 항상 여기를 기준으로 판단한다.

## 0. 단 하나의 분기점
Meta API는 **"내가 권한 가진 계정" vs "남의 공개 계정"** 이 완전히 다르다.

- **내 권한(위임) 계정** → 조회·도달·노출·저장·프로필방문 등 **비공개 인사이트 전부** 합법 확보.
- **남의 공개 계정** → **공개지표(좋아요·댓글수·캡션·해시태그·게시시각)** 만 공식 허용.

본 프로젝트의 주 대상(경쟁사·인플루언서·해시태그)은 대부분 **남의 계정**이다.

## 1. 토큰 = 출입증
사용자가 입력하는 **자기 API 토큰은 Meta Graph API 진입용 "출입증"**.
그 출입증으로 **Business Discovery**를 호출하면 *남의* 공개 비즈니스 계정도 조회 가능.
→ "API 토큰 입력 = 자기 계정만 분석"이 **아니다.**

## 2. 데이터 3분류 (외부 계정 기준)
| 데이터 | 공식 API | 서드파티 | 비고 |
|---|---|---|---|
| 공개지표(좋아요·댓글**수**·캡션·해시태그·시각·미디어타입) | ✅ Business Discovery | ✅ | **Phase 1** |
| 릴스 **재생수(조회수)** | ❌ | ✅ 가능 | 화면에 공개됨 → 수집가능. **Phase 4** |
| 댓글 **내용(텍스트)** | ❌ | ✅ 가능 | 공개됨. **Phase 4** |
| 리포스트/공유 수 | ❌ | △ 일부 | 제한적 |
| **노출/도달(reach·impressions)** | ❌ | ❌ **영영 불가** | 화면에도 안 보이는 비공개값. 소유자만 |

→ **핵심:** "노출"과 "조회"를 혼동 말 것. **조회수는 길이 있고, 노출/도달은 외부 계정에선 어떤 방법으로도 불가.**

## 3. 인스타그램 상세

### 가능 (공식 API)
- **Business Discovery**: 남의 공개 비즈니스/크리에이터 계정 → 팔로워수·게시물수·바이오 + 최근 게시물(캡션, 좋아요수, 댓글수, 미디어타입, 시각, permalink).
  - 분석 가능: 업로드 주기/시간대/빈도, 참여율, 해시태그·키워드, 콘텐츠 포맷 비중, 상대 반응(좋아요·댓글수 기준).
- **Hashtag Search**: 해시태그별 최근/인기 게시물(캡션·좋아요수·댓글수). 한도 있음(아래).
- **Insights (위임 계정만)**: 노출·도달·저장·프로필방문·댓글내용 등 전부.

### 불가 (공식 API)
- 외부 계정의 노출/도달/조회수/댓글내용/리포스트.
- 개인(비즈니스 미전환) 계정은 Business Discovery 대상 아님.

### 콘텐츠 내용 분석 — 강점
- 합법 확보한 캡션 + 미디어 URL에 **AI(Claude) 분석** → 주제·소구점·포맷·카피톤 분류. 차별화 포인트.

## 4. 쓰레드 (Threads) — 보류
- API가 **내 계정 중심**. 내 게시물 발행/인사이트는 ✅.
- 외부 계정 분석: 키워드 검색 일부 외에는 제한적 ⚠️. 서드파티 생태계 미성숙.
- 판정: 인스타 보조 채널 수준. **MVP 제외.**

## 5. 페이스북 — 보류
- 내 페이지 인사이트 ✅.
- 외부 공개 분석: 공개 게시물 검색 폐지, Page Public Content Access 검수 게이트, **CrowdTangle 종료(2024.08)**, Content Library는 학술 전용.
- 판정: 외부 분석 **거의 불가 ❌. MVP 제외** (포함하더라도 "내 페이지 한정").

## 6. 제한량(쿼터/레이트리밋)
| 항목 | 성격 |
|---|---|
| 해시태그 검색 | **하드 쿼터: 토큰(IG 비즈니스 계정)당 7일 30개 고유 태그** |
| 일반 호출(Business Discovery·Insights 등) | **속도 제한**(시간당, 연결 계정수 비례, 넘으면 지연·리셋). 하드 상한 X |
| 토큰 만료 | 장기 토큰 ~60일, 갱신 필요 |
| Business Discovery 범위 | 최근 게시물 중심(아주 오래된 전체 회수는 페이지네이션 한계) |
| 공개 배포 | **앱 검수 + 비즈니스 인증** 필요(고급 액세스). 개발모드는 등록 테스터만 |
| 커버리지 | 상대가 비즈니스/크리에이터 계정일 때만 |

→ 실무 영향: 하드 제한은 해시태그뿐. 나머지는 배치 분산으로 회피. 진짜 관문은 **일반 사용자 오픈 시 앱 검수**.
