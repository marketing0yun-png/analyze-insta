# 01. 프로젝트 개요

## 비전
SNS 트렌드/경쟁 분석기 제품군 중 **Meta 생태계(인스타·쓰레드·페북)** 담당 모듈.
- 자매 모듈: 네이버 생태계(완성), 유튜브(API 가능 확인).
- 본 모듈은 **인스타그램 우선**, 쓰레드 차순위, 페북은 옵션.

## 목적 (우선순위 순)
1. **광고주에게 정확한 정보 제공** — 1차 목적.
2. 경쟁사 루틴 파악 → 벤치마킹 & 경쟁 콘텐츠 발행에 활용.
3. 데이터 축적 → 추후 재가공·고도화.

## 대상 사용자
- **일반 사용자:** 광고 대행/마케터. 자기 Meta API 토큰을 입력해 직접 사용.
- **마스터(프로젝트 오너):** 전체 사용자 데이터를 조합·재가공.

## 핵심 유즈케이스

### UC1. 경쟁 매장 계정 분석
- 입력: 육아용품 매장들의 인스타 계정(username) 목록.
- 산출: 어떻게 홍보하는지 — 업로드 주기/시간대, 참여율, 해시태그 전략, 콘텐츠 포맷·주제.
- 활용: 경쟁사 루틴 체크 → 벤치마킹 → 경쟁 콘텐츠 기획.

### UC2. 해시태그 기반 분석 (보조 기능)
- 입력: 카테고리 해시태그.
- 산출: 해당 태그로 올라온 게시물들의 공개지표(좋아요·댓글수·캡션).
- 한계: 토큰당 7일 30개, 작성자 불명, 조회수 없음. → **보조 기능으로 한정.**
- (원래 원했던 "노출 잘됐나/조회수/댓글반응 깊이"는 외부 계정에선 제약 — `docs/02_CONSTRAINTS.md` 참조.)

### UC3. 인플루언서 계정 분석
- 입력: 육아 인플루언서 계정 목록.
- 산출: UC1과 동일한 분석.

### UC4. 위임 계정 완전 분석 (Phase 3)
- 광고주가 자기 IG 비즈니스 계정 권한을 위임한 경우.
- 산출: 노출·도달·저장·프로필방문 등 **비공개 인사이트 포함** 심층 리포트 + 경쟁 비교.

## 카테고리 개념
- 사용자가 "카테고리"(예: 육아용품)를 지정 → 그 카테고리 맥락에서 계정/해시태그를 묶어 분석.
- 카테고리는 분석 대상(계정·해시태그)을 그룹핑하는 논리적 단위.

## 채널별 적용 범위 (요약 — 상세 `docs/02_CONSTRAINTS.md`)
| 채널 | 외부 분석 | 위임 계정 분석 | 우선순위 |
|---|---|---|---|
| 인스타그램 | ✅ 공개지표 (+서드파티 보강) | ✅ 완전 | **1순위 (집중)** |
| 쓰레드 | ⚠️ 제한적 | ✅ | 보류 |
| 페이스북 | ❌ 거의 불가 | ✅ (내 페이지) | 보류 |
